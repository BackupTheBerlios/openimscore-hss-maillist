Attachment: http://www.openimscore.org/docs/issues/xml-conf.patch (3.46 KB)

I have found that automated testing is complicated by having a
dependency on the filesystem to provide the configuration, especially in
a mixed environment. I suggest adding the option to configure the JDP by
passing the XML as a string.


What I did was replace:
DocumentBuilder builder = factory.newDocumentBuilder();
config = builder.parse( cfgFile );


with:
DocumentBuilder builder = factory.newDocumentBuilder();
config = builder.parse( new InputSource(new StringReader(cfgString)) );


Of course this would change the meaning of the existing parameters
without actually changing the type, which is not a good thing. So I
added a new no-arg constructor and a configure() method that allows the
choice between using a path to an xml file or direct xml. The existing
constructor was rewired to use configure().
