yes, all the SPTs within the same trigger point are combined as following:

1- all the SPTs from the same group (grp field) are combined with the
other SPTs within the same group (inner condition apply)

2-all the SPTs within different groups (outer condition apply)

Where inner condition operator is OR and outer is AND for CNF form. The
opposite is for DNF.

e.g.
for CNF form

trigger_point_1
spt_1 -> grp 0
spt_2 -> grp 0
spt_3 -> grp 1
spt_4 -> grp_2

=>

(spt_1 OR spt_2) AND (spt3) AND (spt_4)

> Hi Rishin,
> 
> Yes, every condition is representing a service point trigger (SPT) in
> the database, which is then linked to the corresponding trigger point.
> The trigger point contains only the name and the form: CNF or DNF.
> (The value for CNF is 1 and DNF is 0.)
> 
> e.g. For CNF this meens that the outer conditions are separated by AND
> and the inner ones by OR. For the DNF the opposite applies.
>
