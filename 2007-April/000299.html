So your request is to explicitly store an S-CSCF name for an IMPU ? We
so far followed the guideline of 29.228 in the DB-revamp branch:

> If there is not any Public User Identity within the IMS Subscription
> with an S-CSCF name assigned, then the HSS shall return the
> Server-Capabilities AVP, which enables the I-CSCF to select an S-CSCF.
> The returned capabilities shall satisfy all the requirements of all
> the service profiles associated with the IMS Subscription. The
> Server-Capabilities AVP may be absent, to indicate to the I-CSCF that
> it may select any available S-CSCF. Experimental-Result-Code shall be
> set to DIAMETER_FIRST_REGISTRATION. The HSS shall not return any
> S-CSCF name.

>From my perspective as well, the HSS should not know too much about
S-CSCF, the I-CSCF should.

- Peter
