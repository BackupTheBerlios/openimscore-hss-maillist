hi Sheik,

It seems that your wireshark trace does not contains the SNR, SNA
messages in it :( Please make another capture. Please use the following
filter when you are capturing:
"
tcp.port == 3868 || tcp.port == 3869 || tcp.port == 3870 || diameter
"
Enter it into the filter tab and then activate it through an enter command.

> Pls tell me how to analyse into these encrypted data using wireshark.

If you had used the filter, then you should see only the relevant
messages. Be aware that not all the diameter messages are automatically
decoded by wireshark. You have to go on them manually and use the decode
tool : decode as Diameter. (right click on a message, decode,
transport->diameter)

Hope it helps.

- -Adrian
