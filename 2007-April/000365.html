Then it seems that eyebeam does not implement the RFC3608 related to
Service-Route. If you still want to use eyebeam, please change the
pcscf.cfg file. Wherever you find if (!P_follows_service_route()){...}
you will need to disable Variant 1, which sends the 400 and enable
Variant 2, which fixes the messages.
