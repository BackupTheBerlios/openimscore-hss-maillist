Well, if you use the OpenIMSCore, you are already part of the mafia ;-).

Please understand that we are not paid directly to develop the Open IMS
Core in the exact way that you want it to be. The Open IMS Core is an
open source project and anyone is welcomed to contribute. As such each
member has its own targets. The benefit is that we are all sharing (or
should be sharing!) our contributions so that others can benefit. This
is the open source model and it does not work too well if just one of
the party is contributing and all the rest benefit without giving
anything back. In any case, you have already contributed because you
reviewed our work and spotted lots and lots of bugs, so you are part of
the team already.

An because "there is no free lunch", everybody is driven in the end by
financial reasons. In FOKUS's case, we are using the Open IMS Core to
promote our IMS expertise and we are also implementing on-demand
features for our customers. To conclude, if anyone needs a features
there are 2 possibilities:
1. They could implement it themselves and contribute it back
2. They could find somebody to implement it for them and pay for it
An option that does not involve spending some sorts of resources does
not exists.

In this particular case of the SessionId AVP, we are not at all troubled
by this and we are sorry, but we can not justify the resources that we
need to spend on providing a solution.

-Dragos
