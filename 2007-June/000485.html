I think that the default auth scheme is on the private identity. So if
each phone has a different private identity it would work and you could
have them all on the same public.

Well, I would have loved to be able to keep the old approach on using
what the client indicated in the Authorization header. But, as you said,
it's kind of impossible to detect well what the client wants. To make
things even worse, for MD5 for example, you have no clue if it means
ETSI, Cable Labs, IETF or FOKUS' MD5...
