What if a user wants to use different terminals with different auth schemes?
Should he use different public identities for the same private id?

IMO the best behavior would be to detect the algorithm parameter in
the first REGISTER.
The problem we have faced is that some clients don't send this
parameter at all in the first REGISTER, so we can't really support
multiple auth schemes at the same time.

JF
