Hi,

I wanted to be able to run the HSS as a daemon on Linux. Below I describe
what I wanted and how I did it. A zip file containing my changes is also
included. The changes are made and tested using revision 455.

I would be happy if you would consider adding this to the trunk. 

Requirements:
1. No output to the screen when running, other than errors during a failed
startup
2. Be able to start, stop and restart using a init.d script
3. Gracefull shutdown of HSS in case Linux is instructed to shutdown

What I did:
I used the guide by Peter Williams at
http://pezra.barelyenough.org/blog/2005/03/java-daemon/

Added shutdown hooks to the HSSConatiner
(/FHoSS/trunk/src/de/fhg/fokus/hss/main/HSSContainer.java) that it will
gracefully shutdown the HSS if the VM recives a SIGTERM signal.

I also changed the configuration of log4j to not include "MYCONSOLE" as a
rootLogger.

And I finally created a bash script for us in init.d. It is written for Red
Hat like systems and tested on Fedora Core 7.

An option for running us daemon would be:
Jakarta Daemon: http://commons.apache.org/daemon/
But it requires the use of native code in C.

Thanks for great software!

Regards,
Mikael Rudholm
