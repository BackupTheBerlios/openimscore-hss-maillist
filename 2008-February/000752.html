Hi David,

according to RFC 3588 the FHoSS probably would not need a Supported-Vendor-Id AVP and for sure no Inband- Security-Id AVP as we don't use TLS per default

<RFC3588>
NO_INBAND_SECURITY                0
      This peer does not support TLS.  This is the default value, if the
      AVP is omitted.
</RFC3588>

In DiameterPeerHSS.xml you set the supported apps, for anything more, feel free to patch the JavaDiameterPeer.

Wasn't your problem the PUR ?

- Peter
