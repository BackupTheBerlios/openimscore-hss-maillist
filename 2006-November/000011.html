Hi Sebastian,

you basically define the SIP URI to be addressed as an application
server in the FHoSS. Once this one is stored, you simply add a trigger
point that should trigger the sending of messages to this SIP AS (see
the default_tp entry in the FHoSS as a reference).

One strange behaviour though is that you have to press "reload" on some
browsers the first time you access the web management console to make
the "create" buttons appear (as hssAdmin) - I guess you were referring
to this ...

Cheers,

Peter
