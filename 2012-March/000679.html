> *Asunto:* DSAIs attached to IMPUs or attached to implicitly registered
> Public User Identity sets?
>
> Hi all,
>
> I have one question dealing with the DSAI concept and the implicitly
> registered Public User Identity sets...
> From my understanding, a DSAI is configured for individual IMPUs, not
> for implicitly registered Public User Identity sets. In such a
> situation, when a DSAI Value is updated for an IMPU, the updated User
> Profile Data has to be sent to the S-CSCF. According to the xml schema
> for the User Data sent through the Cx interface, the SPs for the IMPUs
> that belong to the same set as the IMPU for whom the DSAI Value has
> been changed must be sent.
>
> Please consider the situation:
>
> IMPU1 belongs to an implicitly registered Public User Identity set,
> which is also compound of other IMPUs with the following configuration:
>
> IMPU1: SP1
> IMPU2: SP1
> IMPU3: SP2
>
> These Service Profiles have the next iFCs:
> SP1: iFC1, iFC2
> SP2: iFC3
>
> We have also a DSAI with the following configuration:
> DSAI1
> iFCs associated: iFC1
> IMPUs attached: IMPU1, IMPU2
> DSAI_Value=ACTIVE for both IMPUs, IMPU1 and IMPU2
>
> In this situation, the DSAI_Value is set to INACTIVE for IMPU1, and
> for IMPU2 it keeps being ACTIVE. Then the updated User Data XML has to
> be sent to the S-CSCF through the Cx, and here it comes the problem:
>
>
>  <Objeto OLE: Imagen (Mapa de bits independiente del dispositivo ) >>
>
> What shall be done in such a situation? This could be prevented if we
> attached implicit sets instead of IMPUs to the DSAIs. However the
> specifications talk about IMPUs... Please advice on how to proceed.
> Thanks in advance.
