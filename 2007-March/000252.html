Hi all, 

I have a doubt regarding the sh interface. So, in your application (web interface) you have the AS address as a SIP URI, but according to the 29.328 spec, the AS is identified by the Origin-Host which is an IP address. How do you validate the AS, i.e. how do you check if an AS has permissions to the operation requested? If you only match with the IP part from the SIP URI, there can be a problem because according with your web interface you can have multiple AS with the same IP address (not with the same SIP URI). Which one do you validate?
