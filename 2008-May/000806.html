We are using Rel 7 of FHoSS. We find that PUR Update stores only the last
parameter of Service-Data.
This problem was reported by someone else earlier and the suggestion to fix
was to add a Container around
the data. We added a container and have the same problem.

For example, if the following is sent, 8005551212 is stored in the rep_data
field.

<?xml version="1.0" encoding="UTF-8"?>
<Sh-Data xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
xsi:noNamespaceSchemaLocation="ShDataType.xsd">
<RepositoryData>
<ServiceIndication>AppData</ServiceIndication>
<SequenceNumber>0</SequenceNumber>
<ServiceData>
<Container>
<Name>Test User</Name>
<Phone>8005551212</Phone>
</Container>
</ServiceData>
</RepositoryData>
</Sh-Data>


If instead this is sent, "Test User" is stored.:
<RepositoryData>
<ServiceIndication>AppData</ServiceIndication>
<SequenceNumber>0</SequenceNumber>
<ServiceData>
<Container>
<Name>Test User</Name>
</Container>
</ServiceData>
</RepositoryData>


Finally if this is sent, "8006667676" is stored:
<RepositoryData>
<ServiceIndication>AppData</ServiceIndication>
<SequenceNumber>0</SequenceNumber>
<ServiceData>
<Container>
<Name>Test User</Name>
<Phone>8005551212</Phone>
<Fax>8006667676</Fax>
</Container>
</ServiceData>
</RepositoryData>
</Sh-Data>
