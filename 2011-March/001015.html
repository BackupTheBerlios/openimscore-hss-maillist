Hi Federico,

You can see in TS 29.328 (Table 7.6.1: Data accessible via Sh interface)
that Data-Referecence code 13 (iFC) is not applicable for the Sh-Update
operation, only Sh-Pull and Sh-Subs-Notif, so it is not possible to update
it by using a PUR message. AFAIK, there's no Diameter interface suitable for
managing iFCs, it should be made by other means.
