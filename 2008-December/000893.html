Alexandre Mendonça wrote:
> Hi.
>
> I'm currently integrating Mobicents (http://www.mobicents.org) and 
> OpenIMS and for that, I'm using Sh interface for communication between 
> AS and the HSS.
>
> Something that I've came across was the fact of the PNR being sent by 
> OpenIMS HSS has Auth-Application-Id set to 16777216, which is the 3GPP 
> Cx/Dx interface App Id. Shouldn't it be 16777217 (3GPP Sh) instead?
>
> This is causing me trouble because we don't have Cx/Dx support and so 
> our Diameter Peer rejects the message with 3007 (APPLICATION NOT 
> SUPPORTED) code. Anyway, I've changed the line 103 in PNR.java:
>
> UtilAVP.addVendorSpecificApplicationID(request, 
> DiameterConstants.Vendor.V3GPP, DiameterConstants.Application.Cx 
> <http://DiameterConstants.Application.Cx>);
>
> to
>
> UtilAVP.addVendorSpecificApplicationID(request, 
> DiameterConstants.Vendor.V3GPP, DiameterConstants.Application.Sh 
> <http://DiameterConstants.Application.Sh>);
>
> to make it work, but not sure if that's how it is supposed to be.
