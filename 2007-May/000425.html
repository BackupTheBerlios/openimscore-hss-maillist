Hi Abishek,

the "database" you are writing of maps to the Online Charging Server
(OCS) of 3GPP and is already described in TS 32.260. It allows
monitoring of credit for users and initiates the dropping of calls if
applicable. Via a IMS Gateway Function the SIP messages trigger Diameter
requests to the OCS, but it is not said that this function is part of a
S-CSCF. 

While the work for offline charging support is currently done in the
05_cdp_sessions branch, I currently see that IMS GW Function rather on
application layer than with the S-CSCF. The relation to the HSS is only
the possibility to configure charging endpoints for users in the HSS to
download them during registration. 

Regards,

Peter
