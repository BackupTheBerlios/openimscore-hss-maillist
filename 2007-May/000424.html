> an intern at France Telecom R&D. The project that I'm working on
> consists of prototyping a use case of GUP (General User Profile)
> technology.
> 
> The aim of GUP, which is specified and standardized by 3GPP, is to
> enable harmonized access to user related information. Those
> information
> are distributed over many network elements in a telecom network (HSS,
> AS,  NASS, billing systems, charging systems etc).
> 
> In the GUP architecture, a GUP server stands between IT application
> (clients) and data sources (for example HSS). Two interfaces are
> specified in this architecture, one between client application and GUP
> server (Rg) and another one between GUP server and data sources(Rp). A
> client application has a user profile schema, and can ask GUP server
> for
> a needed piece of information, GUP server knowing where the data
> resides, will look it up and answer the client.
> 
> Those two interfaces are binded to SOAP/XML messages. The main
> operations defined on this interface are :
> *       Creating / delete/ reading/ modifying  a part of user profile
> *       Search operation
> *       Subscribing/unsubscribing for changes of a part of user
> profile
> 
> As far as I know FHoSS server offers Diameter interfaces. I would like
> to ask you if you have considered implementing this SOAP Rp interface
> in
> HSS server? Do you find interesting having such interface? And finally
> could you estimate how difficult would be to implement it.
> 
> Thank you in advance for your comments and suggestions
