Hi Abhishek,

Ok, but how you will specify to the HSS to use that specific table for
your application server (in the case you have more than one AS)? If the
communication between the HSS and the AS is based on Sh interface you
should use the repository data table available. You can define your
specific format for that blob field and add everything you like there.
In my opinion the tables should be as generic as possible and to permit
to all AS to store different kind of data.

However, if you want to make only some testing and is not important for
you what the specs are saying, you can define the table that you want
and to change the current behaviour for UDR and PUR to use your own
specific table.

- -Adrian





Normally the Sh permits you to store AS specific data in a transparent
way, using repository data. There you can add anything you like.
