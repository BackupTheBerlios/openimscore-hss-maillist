is caused by a matching failure between AS name in the HSS and AVP Origin-Host in UDR message. In HSS the AS name is something like this "sip:ip:port", then it is necessary to put the same name in AVP Origin-Host in UDR message including the port part. This seems strange; port part shouldn't be necessary, right?

 

Messages UDA over Sh Interface are returning User-Data AVP with avp code 606. This avp code is valid in Cx interface. In Sh interface it should be 702.

 

Changes in trigger points are not being updated if a registration type (Reg, DeReg and ReReg) change on Trigger Point web interface is not made.

 

These issues are related to rev 280.
