Hi Markus,

> thanks for your answer - your're right. And when operating an HSS it is
> definitively better not to let applications write iFCs... Kind regards,

I remember an argument why this has been avoided during standardisation:
In complex application environment with different AS you sometimes have
the issue of application interaction. This is not easy to be solved (SCIM
and Service Broker ...). And it is totally out of control if an operator
allows to create or modify iFCs. The only accepted activity is to activate
or deactivate an existing iFC using the DSAI.
