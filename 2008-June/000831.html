Hi Markus,
 
> is it possible to modify iFC via the Sh-interface? 

No. 
The only manipulation on iFC by an AS is activation or deactivation via DSAI
(Dynamic Service Activation Info).

> And as far as I understood ETSI TS 129.328 updating a profile with
> new/modified iFCs should be possible.

No. 
If you look at table Table 7.6.1 (Data accessible via Sh interface) you can
see that only Sh-Pull and Sh-Subs-Notif is allowed on this data (no
Sh-Update Operation). That means data can be read by an AS and an AS can get
a notification when iFC s have been changed.
