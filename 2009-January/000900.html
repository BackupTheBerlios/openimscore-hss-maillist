Hi Naveen,

I will try and answer your questions:

UDR/UDA:
1. Are you sure that it is 
DIAMETER_ERROR_IDENTITY_ALREADY_REGISTERED(5005) (inside an Experimental 
Result-Code?) error? In 29.328 and 29.329 I can't find 5005 anywhere, so 
I would guess that you got instead a Result Code of 5005, which based on 
RFC3588 means:

DIAMETER_MISSING_AVP               5005
      The request did not contain an AVP that is required by the Command
      Code definition.  If this value is sent in the Result-Code AVP, a
      Failed-AVP AVP SHOULD be included in the message.  The Failed-AVP
      AVP MUST contain an example of the missing AVP complete with the
      Vendor-Id if applicable.  The value field of the missing AVP
      should be of correct minimum length and contain zeroes.

3. I don't know what the question was... Please send a trace and explain 
what you think that is wrong.

PUR/PUA

1. Have you given the proper access rights to the AS with that FQDN as 
Diameter Name in the HSS web interface?

2. These are values that you are also filling in the web interface. I 
don't think that there is a format enforced in the HSS.

Cheers,
-Dragos
