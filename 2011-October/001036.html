Hi Rocio,

 

I guess you are using a pure SIP client (not IMS aware), which does not
include an Authorization header field in the first REGISTER request.

In that case the IMPI is derived from the IMPU for registration. This is
necessary to get authorization data for the user, otherwise there is no
username available.

 

As far as I remember this is a FOKUS specific authentication method for
Digest-MD5 to help IMS unaware clients to register.
