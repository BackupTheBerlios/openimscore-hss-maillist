Hello Ashish,

Very strange... this problem is persistent or only occurs from time to
time? We have some problems with hibernate, and we are working in
solving them but these occurs only casually and refers to the connectors
 that hibernate use in operating with the database.

If your error is persistent, please test if the hss user (not the root
user!) has the rights in operating with "hssdb". Normally this is
automatically set after you run the sql scripts in the last command
(grant all privileges...) but maybe an unusual operation occurred and
this operation wasn't executed correctly. If hss user has no rights to
operate with the database, the hibernate connection will fail always.
Probably this is your problem.

Cheers,
Adrian
