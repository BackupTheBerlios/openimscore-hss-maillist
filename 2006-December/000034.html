Your exception is surely because of a conflict between the Xerces and
Castor, which is known by the castor team. For solving this I added the
castor.properties file and I set the "namespace" parameter to true.
(This is a solution offered by castor team to this bug). More about this
bug you can found here:
  http://nullpointer.debashish.com/castor-and-the-namespace-bug

Also you should know according to 3GPP specs (TS 23.328) that you can
not update IMSUserState using the Sh interface; you can only to Sh-Pull
or Sh-Subs-Notif.
