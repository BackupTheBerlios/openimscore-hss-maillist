Hi.

It sounds like DNS problems. Review the DNS topics in OpenIMSCore
installation guide.
The IMS nodes won't interoperate until they can resolver their names
between them.
The DNS resolution should be in line with kind of network you are using
(whether loopback or public interface).

For example in my IMS host the icscf.open-ims.test can be resolved to
the public network interface:

        aire at srvimscore:~$ nslookup icscf
        Server:         127.0.0.1
        Address:        127.0.0.1#53
        
        Name:   icscf.open-ims.test
        Address: 192.168.11.3

Do checks with nslookup (or dig, or simple ping) before starting IMS
nodes to ensure that they have visibility between them.

Regards...
