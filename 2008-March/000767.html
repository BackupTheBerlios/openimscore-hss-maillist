There is logically no difference between MD5 authentication scheme proposed
by CableLabs and 3GPP. It is possible (not sure) that CableLabs delta was
submitted to 3GPP and it is incoporated into 3GPP with few addtional changes
(such as change in AVP codes  and use of "SIP Digest" instead of "Digest").
CableLabs spec might get aligned with 3GPP in their next version.

> 2. The new AVPs do not appear in 29.229 8.0.0, so I doubt that the
> vendor id for them is 3GPP...


Please see sections 6.3.37 - 6.3.41 of 29.229 8.0.0.  For these AVPs codes
assigned by 3GPP are based on RFC 4740 (Diameter SIP Application) which
reuses the AVP codes from RFC 4590 (RADIUS Extension for Digest
Authentication. So these AVPs don't have vendor id set. I believe the patch
is in line with this.
