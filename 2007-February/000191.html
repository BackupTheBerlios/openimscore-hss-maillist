Hi Rajaram,

the default handling is per default set to "continued" and should be ok
for all non-blocking services. You configure the forwarding to an AS by
connecting the AS to a trigger point in the initial Filter Criteria.
This way, there are many ways possible to address also more than one AS,
just define a new iFC, attach that to a service profile and that's it.
In case the you want iFC_2 to be executed before iFC_1, you just change
the priority in the service profile.

Regards,

Peter
