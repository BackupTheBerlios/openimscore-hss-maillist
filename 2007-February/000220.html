Hello John,

Are you sure you have provisioned the private identity "test at nw.net"
with the Digest-MD5 algorithm in the HSS web console? I think another
scheme is defined there, so your private identity does not support the
Digest-MD5 scheme requested by the S-CSCF.

You have 2 solutions:
	1 - change the scscf.cfg to use the default authentication scheme
supported by your private identity

	2 - change the authentication scheme supported by that private identity
in the web console (Impi-> AuthenticationScheme) Moreover, you can put
the scheme on "any", so all the schemes will be supported.

Thanks for telling your problem, as another bug was found: it seems that
we do not send the correct response to the S-CSCF, in the MAA response:
as Authentication Scheme not supported error is not sent. This will be
fixed soon :)

Cheers,
Adrian
